### Core Runtime

## 3. Hex Grid Implementation

The hex grid is the foundation of the tactical combat layer. Every coordinate, distance calculation, pathfinding query, and line-of-sight check passes through this module. The implementation follows the flat-top hex orientation (pointy-side-up hexagons arranged in offset rows) using the axial coordinate system as the canonical representation.

### 3.1 Coordinate Systems

**Axial coordinates** `(q, r)` are the storage format. Every hex tile, entity position, and movement target is expressed in axial coordinates. The two axes are not perpendicular -- they are separated by 60 degrees.

**Cube coordinates** `(q, r, s)` add a redundant third axis constrained by `q + r + s = 0`. Many hex algorithms (distance, line drawing, rotation, rings) are simpler and less error-prone in cube form. Convert to cube for algorithms, convert back for storage.

```typescript
// src/hex/HexCoord.ts

/** Axial coordinate -- the canonical storage format. */
export interface AxialCoord {
  readonly q: number;
  readonly r: number;
}

/** Cube coordinate -- used for algorithms. Always satisfies q + r + s === 0. */
export interface CubeCoord {
  readonly q: number;
  readonly r: number;
  readonly s: number;
}

/** Convert axial to cube. */
export function axialToCube(a: AxialCoord): CubeCoord {
  return { q: a.q, r: a.r, s: -a.q - a.r };
}

/** Convert cube to axial (drop s). */
export function cubeToAxial(c: CubeCoord): AxialCoord {
  return { q: c.q, r: c.r };
}

/**
 * Produce a stable string key for use in Map/Set.
 * Format: "q,r" -- compact, fast to parse.
 */
export function hexKey(q: number, r: number): string {
  return `${q},${r}`;
}
```

### 3.2 Hex Layout: Hex-to-Pixel and Pixel-to-Hex

The layout defines how axial coordinates map to screen-space (Babylon.js world-space) positions. We use **flat-top** orientation with the following forward matrix:

```
x = size * (sqrt(3) * q  +  sqrt(3)/2 * r)
y = size * (           0  +       3/2  * r)
```

Where `size` is the distance from the center of a hex to any vertex (the outer radius). The inner radius (center to edge midpoint) is `size * sqrt(3) / 2`.

```typescript
// src/hex/HexLayout.ts

const SQRT3 = Math.sqrt(3);

export interface HexLayout {
  /** Outer radius: center to vertex. */
  readonly size: number;
  /** Pixel offset of the grid origin. */
  readonly originX: number;
  readonly originY: number;
}

export function createLayout(size: number, originX = 0, originY = 0): HexLayout {
  return { size, originX, originY };
}

/** Convert axial (q, r) to pixel (x, y). */
export function hexToPixel(layout: HexLayout, q: number, r: number): { x: number; y: number } {
  const x = layout.size * (SQRT3 * q + (SQRT3 / 2) * r) + layout.originX;
  const y = layout.size * ((3 / 2) * r) + layout.originY;
  return { x, y };
}

/**
 * Convert pixel (x, y) to fractional axial coordinates.
 * The result must be rounded to find the nearest hex.
 */
export function pixelToFractionalHex(
  layout: HexLayout,
  px: number,
  py: number
): { q: number; r: number } {
  const x = px - layout.originX;
  const y = py - layout.originY;
  const q = ((SQRT3 / 3) * x - (1 / 3) * y) / layout.size;
  const r = ((2 / 3) * y) / layout.size;
  return { q, r };
}

/**
 * Round fractional axial coordinates to the nearest integer hex.
 * Works by converting to fractional cube, rounding each component,
 * then correcting the component with the largest rounding error.
 */
export function hexRound(fq: number, fr: number): AxialCoord {
  const fs = -fq - fr;
  let q = Math.round(fq);
  let r = Math.round(fr);
  let s = Math.round(fs);

  const qDiff = Math.abs(q - fq);
  const rDiff = Math.abs(r - fr);
  const sDiff = Math.abs(s - fs);

  if (qDiff > rDiff && qDiff > sDiff) {
    q = -r - s;
  } else if (rDiff > sDiff) {
    r = -q - s;
  }
  // else s = -q - r (implicit, we only store q and r)

  return { q, r };
}

/** Full pixel-to-hex: convert pixel to fractional, then round. */
export function pixelToHex(layout: HexLayout, px: number, py: number): AxialCoord {
  const frac = pixelToFractionalHex(layout, px, py);
  return hexRound(frac.q, frac.r);
}
```

### 3.3 Distance, Neighbors, and Rings

```typescript
// src/hex/HexMath.ts

import type { AxialCoord, CubeCoord } from "./HexCoord";
import { axialToCube } from "./HexCoord";

/**
 * Hex distance (minimum number of steps between two hexes).
 * Uses the cube-coordinate formula: max(|dq|, |dr|, |ds|)
 * which is equivalent to (|dq| + |dr| + |ds|) / 2.
 */
export function hexDistance(a: AxialCoord, b: AxialCoord): number {
  const ac = axialToCube(a);
  const bc = axialToCube(b);
  return Math.max(
    Math.abs(ac.q - bc.q),
    Math.abs(ac.r - bc.r),
    Math.abs(ac.s - bc.s)
  );
}

/**
 * The six axial direction vectors, indexed 0-5 starting from east,
 * proceeding counter-clockwise.
 *
 *   0: (+1,  0)  East
 *   1: (+1, -1)  Northeast
 *   2: ( 0, -1)  Northwest
 *   3: (-1,  0)  West
 *   4: (-1, +1)  Southwest
 *   5: ( 0, +1)  Southeast
 */
export const HEX_DIRECTIONS: readonly AxialCoord[] = [
  { q: +1, r:  0 },
  { q: +1, r: -1 },
  { q:  0, r: -1 },
  { q: -1, r:  0 },
  { q: -1, r: +1 },
  { q:  0, r: +1 },
] as const;

/** Return the neighbor of (q, r) in the given direction (0-5). */
export function hexNeighbor(q: number, r: number, direction: number): AxialCoord {
  const d = HEX_DIRECTIONS[direction];
  return { q: q + d.q, r: r + d.r };
}

/** Return all six neighbors. */
export function hexNeighbors(q: number, r: number): AxialCoord[] {
  return HEX_DIRECTIONS.map((d) => ({ q: q + d.q, r: r + d.r }));
}

/**
 * Return all hexes exactly `radius` steps away (a hex ring).
 * Start at the hex `radius` steps in direction 4 (southwest),
 * then walk around the ring in 6 segments.
 */
export function hexRing(center: AxialCoord, radius: number): AxialCoord[] {
  if (radius === 0) return [{ q: center.q, r: center.r }];

  const results: AxialCoord[] = [];
  // Start position: move `radius` steps in direction 4
  let q = center.q + HEX_DIRECTIONS[4].q * radius;
  let r = center.r + HEX_DIRECTIONS[4].r * radius;

  for (let side = 0; side < 6; side++) {
    for (let step = 0; step < radius; step++) {
      results.push({ q, r });
      const dir = HEX_DIRECTIONS[side];
      q += dir.q;
      r += dir.r;
    }
  }
  return results;
}

/** Return all hexes within `radius` steps (a filled hex area). */
export function hexSpiral(center: AxialCoord, radius: number): AxialCoord[] {
  const results: AxialCoord[] = [{ q: center.q, r: center.r }];
  for (let r = 1; r <= radius; r++) {
    results.push(...hexRing(center, r));
  }
  return results;
}
```

### 3.4 Hex Grid Data Structure

```typescript
// src/hex/HexGrid.ts

import type { AxialCoord } from "./HexCoord";
import { hexKey } from "./HexCoord";

export enum TerrainType {
  Grass = "grass",
  Forest = "forest",
  Swamp = "swamp",
  Hills = "hills",
  Mountains = "mountains",
  Sand = "sand",
  Snow = "snow",
  Water = "water",
  Road = "road",
}

export interface HexTile {
  readonly q: number;
  readonly r: number;
  /** Height level 0-3. Affects combat bonuses and movement cost. */
  elevation: number;
  terrain: TerrainType;
  /** Entity ID occupying this tile, or null. */
  occupant: string | null;
  /** True if the tile blocks line-of-sight (e.g., tall forest, mountain). */
  blocksLoS: boolean;
  /** Movement cost multiplier (1.0 = normal, 2.0 = difficult terrain). */
  movementCost: number;
  /** Defense bonus granted to occupant (+10, +15, etc.). */
  defenseBonusMelee: number;
  defenseBonusRanged: number;
}

export class HexGrid {
  private tiles = new Map<string, HexTile>();

  get size(): number {
    return this.tiles.size;
  }

  /** Add or overwrite a tile. */
  set(q: number, r: number, tile: HexTile): void {
    this.tiles.set(hexKey(q, r), tile);
  }

  /** Retrieve a tile, or undefined if out of bounds. */
  get(q: number, r: number): HexTile | undefined {
    return this.tiles.get(hexKey(q, r));
  }

  has(q: number, r: number): boolean {
    return this.tiles.has(hexKey(q, r));
  }

  /** Iterate all tiles. */
  values(): IterableIterator<HexTile> {
    return this.tiles.values();
  }

  /** Return all tiles as an array (useful for rendering). */
  toArray(): HexTile[] {
    return [...this.tiles.values()];
  }
}
```

### 3.5 A* Pathfinding on the Hex Grid

The pathfinder operates on axial coordinates, queries the `HexGrid` for terrain costs and occupancy, and accounts for elevation changes. The heuristic is hex distance (admissible because hex distance equals the minimum number of steps).

```typescript
// src/hex/HexPathfinding.ts

import type { AxialCoord } from "./HexCoord";
import { hexKey } from "./HexCoord";
import { hexDistance, hexNeighbors } from "./HexMath";
import type { HexGrid, HexTile } from "./HexGrid";

interface PathNode {
  q: number;
  r: number;
  g: number; // cost from start
  f: number; // g + heuristic
  parent: PathNode | null;
}

export interface PathResult {
  /** Ordered list of hexes from start (exclusive) to goal (inclusive). */
  path: AxialCoord[];
  /** Total movement cost. */
  cost: number;
  /** True if the goal was reachable. */
  found: boolean;
}

/**
 * Calculate the movement cost to enter `target` from `source`.
 * Returns Infinity if the tile is impassable.
 */
function stepCost(grid: HexGrid, source: HexTile, target: HexTile): number {
  // Impassable tiles
  if (target.terrain === "water" || target.terrain === "mountains") return Infinity;
  // Occupied by another entity
  if (target.occupant !== null) return Infinity;

  let cost = target.movementCost; // base terrain cost

  // Elevation change cost: going up costs +1 per level, going down is free
  const elevDiff = target.elevation - source.elevation;
  if (elevDiff > 0) {
    cost += elevDiff; // climbing penalty
  }

  return cost;
}

/**
 * A* pathfinding from `start` to `goal`.
 * `maxCost` limits the search to a movement budget (e.g., the unit's AP).
 */
export function findPath(
  grid: HexGrid,
  start: AxialCoord,
  goal: AxialCoord,
  maxCost = Infinity
): PathResult {
  const openSet: PathNode[] = [];
  const closedSet = new Set<string>();
  const gScores = new Map<string, number>();

  const startKey = hexKey(start.q, start.r);
  const goalKey = hexKey(goal.q, goal.r);

  const startNode: PathNode = {
    q: start.q,
    r: start.r,
    g: 0,
    f: hexDistance(start, goal),
    parent: null,
  };
  openSet.push(startNode);
  gScores.set(startKey, 0);

  while (openSet.length > 0) {
    // Find node with lowest f (use a binary heap in production for performance)
    openSet.sort((a, b) => a.f - b.f);
    const current = openSet.shift()!;
    const currentKey = hexKey(current.q, current.r);

    if (currentKey === goalKey) {
      // Reconstruct path
      const path: AxialCoord[] = [];
      let node: PathNode | null = current;
      while (node !== null && !(node.q === start.q && node.r === start.r)) {
        path.unshift({ q: node.q, r: node.r });
        node = node.parent;
      }
      return { path, cost: current.g, found: true };
    }

    closedSet.add(currentKey);
    const currentTile = grid.get(current.q, current.r);
    if (!currentTile) continue;

    for (const neighbor of hexNeighbors(current.q, current.r)) {
      const nKey = hexKey(neighbor.q, neighbor.r);
      if (closedSet.has(nKey)) continue;

      const neighborTile = grid.get(neighbor.q, neighbor.r);
      if (!neighborTile) continue;

      const cost = stepCost(grid, currentTile, neighborTile);
      if (cost === Infinity) continue;

      const tentativeG = current.g + cost;
      if (tentativeG > maxCost) continue;

      const prevG = gScores.get(nKey) ?? Infinity;
      if (tentativeG >= prevG) continue;

      gScores.set(nKey, tentativeG);

      const neighborNode: PathNode = {
        q: neighbor.q,
        r: neighbor.r,
        g: tentativeG,
        f: tentativeG + hexDistance(neighbor, goal),
        parent: current,
      };
      openSet.push(neighborNode);
    }
  }

  return { path: [], cost: 0, found: false };
}

/**
 * Return all hexes reachable within `budget` movement cost.
 * Used to highlight the movement range overlay.
 */
export function reachableHexes(
  grid: HexGrid,
  start: AxialCoord,
  budget: number
): Map<string, number> {
  const visited = new Map<string, number>(); // key -> cost to reach
  const frontier: Array<{ q: number; r: number; cost: number }> = [];

  visited.set(hexKey(start.q, start.r), 0);
  frontier.push({ q: start.q, r: start.r, cost: 0 });

  while (frontier.length > 0) {
    // Priority queue by cost (use a heap in production)
    frontier.sort((a, b) => a.cost - b.cost);
    const current = frontier.shift()!;
    const currentTile = grid.get(current.q, current.r);
    if (!currentTile) continue;

    for (const neighbor of hexNeighbors(current.q, current.r)) {
      const nKey = hexKey(neighbor.q, neighbor.r);
      const neighborTile = grid.get(neighbor.q, neighbor.r);
      if (!neighborTile) continue;

      const cost = stepCost(grid, currentTile, neighborTile);
      if (cost === Infinity) continue;

      const totalCost = current.cost + cost;
      if (totalCost > budget) continue;

      const prevCost = visited.get(nKey) ?? Infinity;
      if (totalCost >= prevCost) continue;

      visited.set(nKey, totalCost);
      frontier.push({ q: neighbor.q, r: neighbor.r, cost: totalCost });
    }
  }

  return visited;
}
```

### 3.6 Line of Sight

Line of sight uses the hex line drawing algorithm. A straight line is drawn from the center of the source hex to the center of the target hex in cube coordinate space, sampling at regular intervals and rounding each sample to the nearest hex. If any intermediate hex has `blocksLoS === true`, the line of sight is blocked.

```typescript
// src/hex/HexLineOfSight.ts

import type { AxialCoord, CubeCoord } from "./HexCoord";
import { axialToCube, cubeToAxial } from "./HexCoord";
import { hexDistance } from "./HexMath";
import type { HexGrid } from "./HexGrid";

function lerp(a: number, b: number, t: number): number {
  return a + (b - a) * t;
}

function cubeLerp(a: CubeCoord, b: CubeCoord, t: number): CubeCoord {
  return {
    q: lerp(a.q, b.q, t),
    r: lerp(a.r, b.r, t),
    s: lerp(a.s, b.s, t),
  };
}

function cubeRound(q: number, r: number, s: number): CubeCoord {
  let rq = Math.round(q);
  let rr = Math.round(r);
  let rs = Math.round(s);

  const dq = Math.abs(rq - q);
  const dr = Math.abs(rr - r);
  const ds = Math.abs(rs - s);

  if (dq > dr && dq > ds) {
    rq = -rr - rs;
  } else if (dr > ds) {
    rr = -rq - rs;
  } else {
    rs = -rq - rr;
  }

  return { q: rq, r: rr, s: rs };
}

/**
 * Draw a hex line from `a` to `b`.
 * Returns every hex along the line, including start and end.
 * Nudges the start slightly to avoid ambiguous midpoint snapping.
 */
export function hexLineDraw(a: AxialCoord, b: AxialCoord): AxialCoord[] {
  const ac = axialToCube(a);
  const bc = axialToCube(b);
  const dist = hexDistance(a, b);
  if (dist === 0) return [{ q: a.q, r: a.r }];

  // Nudge to avoid ties at edge midpoints
  const nudgedA: CubeCoord = {
    q: ac.q + 1e-6,
    r: ac.r + 1e-6,
    s: ac.s - 2e-6,
  };
  const nudgedB: CubeCoord = {
    q: bc.q + 1e-6,
    r: bc.r + 1e-6,
    s: bc.s - 2e-6,
  };

  const results: AxialCoord[] = [];
  for (let i = 0; i <= dist; i++) {
    const t = i / dist;
    const lerped = cubeLerp(nudgedA, nudgedB, t);
    const rounded = cubeRound(lerped.q, lerped.r, lerped.s);
    results.push(cubeToAxial(rounded));
  }
  return results;
}

/**
 * Check line of sight from `source` to `target`.
 * Returns true if the line is clear (no blocking hexes between them).
 * The source and target hexes themselves are NOT checked for blocking.
 *
 * Elevation rule: a hex blocks LoS only if its elevation is >= both
 * the source elevation and the target elevation.
 */
export function hasLineOfSight(
  grid: HexGrid,
  source: AxialCoord,
  target: AxialCoord
): boolean {
  const line = hexLineDraw(source, target);

  const sourceTile = grid.get(source.q, source.r);
  const targetTile = grid.get(target.q, target.r);
  if (!sourceTile || !targetTile) return false;

  const sourceElev = sourceTile.elevation;
  const targetElev = targetTile.elevation;

  // Check intermediate hexes (skip first = source, skip last = target)
  for (let i = 1; i < line.length - 1; i++) {
    const hex = line[i];
    const tile = grid.get(hex.q, hex.r);
    if (!tile) return false; // off-grid = blocked

    if (tile.blocksLoS && tile.elevation >= sourceElev && tile.elevation >= targetElev) {
      return false;
    }
  }

  return true;
}
```

### 3.7 Field of View (Shadow Casting for Hex)

For computing which tiles are visible to a unit (fog of war, ranged attack targeting), we use an adapted recursive shadow casting algorithm. The algorithm works in 6 sextants (one per hex direction) and casts shadows from blocking tiles outward.

```typescript
// src/hex/HexFieldOfView.ts

import type { AxialCoord } from "./HexCoord";
import { hexKey } from "./HexCoord";
import { hexDistance } from "./HexMath";
import { hasLineOfSight } from "./HexLineOfSight";
import type { HexGrid } from "./HexGrid";

/**
 * Compute the set of hexes visible from `origin` within `maxRange`.
 *
 * This implementation uses a simple raycasting approach: cast a line from
 * the origin to every hex on the perimeter ring at maxRange. Each hex
 * along the line that has LoS from the origin is marked visible.
 *
 * For maps up to radius 12-15 (typical tactical combat), this performs
 * well enough. For larger maps, switch to proper hex shadow casting.
 */
export function computeFieldOfView(
  grid: HexGrid,
  origin: AxialCoord,
  maxRange: number
): Set<string> {
  const visible = new Set<string>();
  visible.add(hexKey(origin.q, origin.r));

  // For each hex within range, check LoS
  for (let dq = -maxRange; dq <= maxRange; dq++) {
    for (let dr = -maxRange; dr <= maxRange; dr++) {
      const ds = -dq - dr;
      if (Math.max(Math.abs(dq), Math.abs(dr), Math.abs(ds)) > maxRange) continue;

      const target: AxialCoord = { q: origin.q + dq, r: origin.r + dr };
      if (!grid.has(target.q, target.r)) continue;

      if (hasLineOfSight(grid, origin, target)) {
        visible.add(hexKey(target.q, target.r));
      }
    }
  }

  return visible;
}
```

### 3.8 Elevation Model

Every hex tile carries an `elevation` property (integer 0-3). Elevation affects three systems:

1. **Movement cost**: Moving uphill costs +1 AP per elevation level gained. Moving downhill is free. Moving from elevation 2 to elevation 0 costs only the base terrain cost. Moving from elevation 0 to elevation 2 costs base terrain cost + 2.

2. **Combat bonuses**: A unit on higher ground than its target gains +10% melee hit chance and +10% ranged hit chance per elevation level difference. A unit on lower ground suffers the inverse penalty.

3. **Line of sight**: An intervening hex blocks LoS only if its elevation is at least as high as both the source and target hex elevations. This means units on hilltops can see over forests in valleys.

### 3.9 Reference

The hex grid implementation is based on the comprehensive guide by Red Blob Games: https://www.redblobgames.com/grids/hexagons/. All coordinate systems, conversion formulas, distance calculations, and line-drawing algorithms follow the conventions documented there.

## 4. Game State Machine

The game uses a hierarchical finite state machine (HFSM). Each state has `onEnter()`, `onUpdate(dt)`, and `onExit()` lifecycle hooks. States can contain sub-states. Only the deepest active state receives `onUpdate()` calls, but `onEnter()` and `onExit()` cascade through the hierarchy.

### 4.1 State Machine Implementation

```typescript
// src/core/StateMachine.ts

export interface State {
  readonly name: string;
  onEnter?(): void;
  onUpdate?(dt: number): void;
  onExit?(): void;
}

export class StateMachine {
  private states = new Map<string, State>();
  private stack: State[] = []; // supports push/pop for overlay states
  private current: State | null = null;

  register(state: State): void {
    this.states.set(state.name, state);
  }

  /** Transition to a new state, replacing the current one. */
  transition(name: string): void {
    const next = this.states.get(name);
    if (!next) throw new Error(`Unknown state: ${name}`);

    this.current?.onExit?.();
    this.current = next;
    this.current.onEnter?.();
  }

  /** Push a state on top (for overlays like inventory). */
  push(name: string): void {
    const next = this.states.get(name);
    if (!next) throw new Error(`Unknown state: ${name}`);

    if (this.current) this.stack.push(this.current);
    this.current?.onExit?.();
    this.current = next;
    this.current.onEnter?.();
  }

  /** Pop back to the previous state. */
  pop(): void {
    this.current?.onExit?.();
    this.current = this.stack.pop() ?? null;
    this.current?.onEnter?.();
  }

  update(dt: number): void {
    this.current?.onUpdate?.(dt);
  }

  get currentState(): string | null {
    return this.current?.name ?? null;
  }
}
```

```typescript
// src/entities/World.ts

import type { EntityId } from "./Entity";
import type { Component } from "./Component";
import type { System } from "./System";

export class World {
  private entities = new Set<EntityId>();
  private components = new Map<string, Map<EntityId, Component>>();
  // componentType -> (entityId -> component)
  private systems: System[] = [];
  private nextId = 0;

  createEntity(): EntityId {
    const id = `e${this.nextId++}`;
    this.entities.add(id);
    return id;
  }

  destroyEntity(id: EntityId): void {
    this.entities.delete(id);
    for (const store of this.components.values()) {
      store.delete(id);
    }
  }

  addComponent<T extends Component>(entityId: EntityId, component: T): void {
    let store = this.components.get(component.type);
    if (!store) {
      store = new Map();
      this.components.set(component.type, store);
    }
    store.set(entityId, component);
  }

  getComponent<T extends Component>(entityId: EntityId, type: string): T | undefined {
    return this.components.get(type)?.get(entityId) as T | undefined;
  }

  /** Query all entities that have ALL the specified component types. */
  query(...types: string[]): EntityId[] {
    const result: EntityId[] = [];
    for (const id of this.entities) {
      const hasAll = types.every((t) => this.components.get(t)?.has(id));
      if (hasAll) result.push(id);
    }
    return result;
  }

  registerSystem(system: System): void {
    this.systems.push(system);
  }

  update(dt: number): void {
    for (const system of this.systems) {
      system.update(this, dt);
    }
  }

  /** Serialize all entities and components for saving. */
  serialize(): object {
    const data: Record<string, Record<string, Component>> = {};
    for (const id of this.entities) {
      data[id] = {};
      for (const [type, store] of this.components) {
        const comp = store.get(id);
        if (comp) data[id][type] = comp;
      }
    }
    return data;
  }
}
```

### 5.4 Key Systems

**MovementSystem**: Processes move actions. Reads `Position`, writes `Position`, reads/writes `Fatigue`. Validates movement cost against remaining AP. Updates the `HexGrid` occupant references.

**CombatSystem**: Processes attack actions. Delegates to `MeleeResolver` or `RangedResolver` depending on weapon type. Reads `Stats`, `Equipment`, `Position`, writes `Health`, `Armor`, `StatusEffects`.

**MoraleSystem**: Checks morale triggers at the end of each action (ally killed, surrounded, etc.). Reads `Morale`, `Health`, writes `Morale`. When morale drops to "fleeing", the unit's next turn is forced to move away from enemies.

**AISystem**: Runs during `EnemyTurn`. For each entity with `AIBehavior`, evaluates possible actions using a utility scoring system, picks the highest-scoring action, and enqueues it.

**RenderSystem**: Each frame, iterates all entities with `Position` and `SpriteRef`, updates sprite positions to match hex-to-pixel conversion. Handles sprite animation frame advancement.

**FatigueSystem**: At the start of each entity's turn, reduces current fatigue by `recoveryPerTurn`. Caps fatigue at 0 minimum.

**InitiativeSystem**: At the start of each round, recalculates `effective` initiative for all entities: `base - currentFatigue + sum(modifier bonuses from StatusEffects and Equipment)`. The turn order is then sorted by effective initiative (highest goes first).

**StatusEffectSystem**: At the start of each entity's turn, ticks all active status effects: decrement `remainingTurns`, apply per-turn effects (e.g., poison damage, regeneration), remove effects with 0 remaining turns.

## 6. Combat Resolution Engine

### 6.1 Turn Order

At the start of each round, compute the effective initiative for every living combatant:

```
effectiveInitiative = baseInitiative
                    - currentFatigue
                    + equipmentModifier
                    + statusEffectModifier
                    + perkModifier
                    + randomTiebreaker (0-5, seeded)
```

Sort all combatants descending by effective initiative. This order is displayed in the turn order bar at the top of the combat UI. When a unit has the "Wait" action, it moves to the end of the current round's order.

### 6.2 Action Queue

Each unit's turn follows a pipeline:

```
1. Select Action (player choice or AI decision)
2. Validate Action
   - Does the unit have enough AP?
   - Is the target in range?
   - Is the target valid (alive, correct tile)?
3. Commit Action (point of no return)
4. Animate Action (visual feedback, non-blocking to game logic)
5. Resolve Action (apply damage, status effects, triggers)
6. Post-Resolution Checks
   - Did the target die? -> remove from grid, trigger morale checks
   - Did morale break? -> apply fleeing state
   - Any on-hit/on-kill perk triggers?
7. Advance to next unit or next round
```

### 6.3 Melee Attack Resolution

```typescript
// Pseudocode for melee attack resolution

function resolveMeleeAttack(attacker: Entity, defender: Entity, weapon: WeaponData): AttackResult {
  // 1. Calculate hit chance
  let hitChance = attacker.stats.meleeSkill
                - defender.stats.meleeDefense
                + weapon.hitBonus
                + elevationBonus(attacker, defender)  // +10% per level above
                + flanking bonus                      // +10% per ally adjacent to defender (beyond first)
                + statusModifiers(attacker, "meleeSkill")
                + statusModifiers(defender, "meleeDefense")
                ;

  // Clamp to [5%, 95%] -- always a chance to hit or miss
  hitChance = clamp(hitChance, 5, 95);

  // 2. Roll to hit
  const roll = rng.nextInt(1, 100);
  if (roll > hitChance) {
    return { hit: false, damage: 0, armorDamage: 0 };
  }

  // 3. Roll damage
  const baseDamage = rng.nextInt(weapon.damageMin, weapon.damageMax);

  // 4. Apply damage to armor first
  let remainingDamage = baseDamage;
  let armorDamage = 0;

  if (defender.armor.body) {
    const armorAbsorb = Math.min(
      defender.armor.body.currentDurability,
      Math.floor(remainingDamage * (1 - weapon.armorPiercing))
    );
    defender.armor.body.currentDurability -= armorAbsorb + Math.floor(remainingDamage * weapon.armorEffectiveness);
    armorDamage = armorAbsorb;
    remainingDamage -= armorAbsorb;
  }

  // 5. Apply remaining damage to HP
  const hpDamage = Math.max(0, remainingDamage);
  defender.health.current -= hpDamage;

  // 6. Check for injury (if single hit > 15% of max HP)
  if (hpDamage > defender.health.max * 0.15) {
    const injury = rollInjury(weapon.type); // weapon type influences injury table
    defender.health.injuries.push(injury);
    applyInjuryEffects(defender, injury);
  }

  // 7. Fatigue cost for attacker
  attacker.fatigue.current += weapon.fatigueCost;

  // 8. Return result for animation system
  return {
    hit: true,
    damage: hpDamage,
    armorDamage,
    injury: hpDamage > defender.health.max * 0.15 ? injury : null,
    killed: defender.health.current <= 0,
  };
}
```

### 6.4 Ranged Attack Resolution

Similar to melee, but with additional modifiers:

- **Distance penalty**: -10% hit chance per hex beyond optimal range.
- **Scatter**: Missed ranged attacks may hit an adjacent hex (20% chance per neighbor of the target that is occupied by a different entity).
- **Obstructed shot**: If any hex between attacker and target is occupied by another entity, -20% hit chance (shooting through friendly or enemy lines).
- **High ground**: +15% per elevation level above target (ranged benefits more from elevation than melee).

### 6.5 Morale Checks

Morale checks occur on the following triggers:

| Trigger | Morale Change |
|---------|--------------|
| Ally killed nearby (within 3 hexes) | -8 to -15 depending on resolve |
| Ally killed (anywhere) | -3 to -5 |
| Leader killed | -20 to all allies |
| Surrounded (3+ enemies adjacent) | -5 per turn while surrounded |
| Enemy killed nearby | +3 to +5 |
| Taking a heavy hit (>25% max HP) | -5 to -10 |
| Battle turning (>50% of party dead) | -10 once |
| Confident (>50% of enemies dead) | +5 once |

When morale drops below thresholds:
- **Wavering** (below 50): -10% to all attack skills.
- **Breaking** (below 30): -20% to all stats, may fail to execute actions (25% chance to cower instead).
- **Fleeing** (below 10): Unit turns are forced to move away from nearest enemy. Unit cannot attack.
- **Rallying**: At the start of each turn, if morale is below "steady" (70), roll resolve check. On success, morale increases by 10-20.

## 7. Mobile Optimization

### 7.1 Draw Call Reduction

Draw calls are the primary performance bottleneck on mobile GPUs. The target is fewer than 50 draw calls per frame during combat.

**Hex Tiles -- ThinInstances**: All hex tiles of the same terrain type share a single base mesh and use Babylon.js `ThinInstances`. A single draw call renders all grass hexes, another renders all forest hexes, etc. With 6-8 terrain types visible at once, this is 6-8 draw calls for the entire battlefield floor.

```typescript
// Example: ThinInstance setup for hex tiles
const grassMesh = MeshBuilder.CreateDisc("grass", { radius: hexSize, tessellation: 6 }, scene);
grassMesh.rotation.y = Math.PI / 6; // flat-top orientation

const grassMaterial = new StandardMaterial("grassMat", scene);
grassMaterial.diffuseTexture = new Texture("assets/tiles/terrain/grass.png", scene);
grassMaterial.freeze(); // material will not change

// For each grass tile, add a ThinInstance with a position matrix
const matrices: Matrix[] = [];
for (const tile of grid.toArray().filter(t => t.terrain === TerrainType.Grass)) {
  const { x, y } = hexToPixel(layout, tile.q, tile.r);
  matrices.push(Matrix.Translation(x, tile.elevation * elevationStep, y));
}
grassMesh.thinInstanceSetBuffer("matrix", Float32Array.from(matrices.flatMap(m => m.toArray())));
```

**Sprite Atlases**: All character sprites (idle, walk, attack, death frames) are packed into texture atlases (one per faction or unit class). The `SpriteManager` in Babylon.js handles batched sprite rendering.

**Mesh Merging**: Static UI elements or decorative props that share a material are merged with `Mesh.MergeMeshes()` at load time.

### 7.2 Texture Optimization

- Maximum texture size: **2048x2048** pixels. No texture exceeds this. Most UI elements use **1024x1024** or smaller.
- Format: **KTX2 with Basis Universal** compression. Babylon.js has built-in support via `@babylonjs/core/Misc/basis`. This reduces GPU memory 4-8x compared to uncompressed RGBA.
- Mipmaps: Enabled for terrain tiles and sprites. Disabled for UI textures (always rendered at native resolution).
- Texture reuse: All units of the same type share the same sprite atlas. Tinting via vertex color differentiates factions.

### 7.6 Frame Rate Target

- **Target**: 30-60 FPS on mid-range phones (circa 2020 and newer: Snapdragon 720G, Apple A13, equivalent).
- **Strategy**: The game loop uses a fixed timestep for logic (16.67 ms = 60 Hz) and a variable timestep for rendering. If the frame rate drops below 30 FPS for more than 2 seconds, the hardware scaling level is automatically increased by 0.25 until the target is met.

### 7.7 Asset Loading Strategy

- **Lazy loading by scene**: Overworld and combat use different asset sets. When transitioning from overworld to combat, the overworld rendering assets are disposed and combat assets are loaded. This keeps peak GPU memory low.
- **Progressive loading**: Non-critical assets (music, ambient effects, high-res decorations) are loaded after the core scene is interactive. A lightweight loading indicator shows progress.
- **Preloading hints**: During deployment phase, preload enemy sprite atlases based on the enemy roster.

## 8. Touch Input System

### 8.1 Gesture Recognition

All input comes through touch events on the canvas. The input system recognizes the following gestures:

| Gesture | Detection | Game Action |
|---------|-----------|------------|
| **Tap** | Touch down + up within 200ms, movement < 10px | Select hex, select UI element, confirm action |
| **Double-tap** | Two taps within 300ms on the same hex | Center camera on hex, open unit detail panel |
| **Long-press** | Touch held > 500ms without movement | Show tooltip (unit stats, terrain info) |
| **Drag** | Touch down + move > 10px | Pan the camera |
| **Pinch** | Two touches, distance changes > 10px | Zoom the camera (adjust ortho size) |
| **Two-finger drag** | Two touches, same direction | Reserved (unused currently) |

```typescript
// src/core/TouchInput.ts

export interface GestureEvent {
  type: "tap" | "double-tap" | "long-press" | "drag" | "pinch";
  screenX: number;
  screenY: number;
  // Pinch-specific
  scale?: number;       // relative scale factor
  // Drag-specific
  deltaX?: number;
  deltaY?: number;
}

export class TouchInputManager {
  private canvas: HTMLCanvasElement;
  private listeners: Array<(event: GestureEvent) => void> = [];
  private touchStart: { x: number; y: number; time: number } | null = null;
  private lastTap: { x: number; y: number; time: number } | null = null;
  private longPressTimer: ReturnType<typeof setTimeout> | null = null;
  private initialPinchDistance: number | null = null;

  constructor(canvas: HTMLCanvasElement) {
    this.canvas = canvas;
    canvas.addEventListener("touchstart", this.onTouchStart.bind(this), { passive: false });
    canvas.addEventListener("touchmove", this.onTouchMove.bind(this), { passive: false });
    canvas.addEventListener("touchend", this.onTouchEnd.bind(this), { passive: false });
    canvas.addEventListener("touchcancel", this.onTouchCancel.bind(this));
  }

  onGesture(callback: (event: GestureEvent) => void): void {
    this.listeners.push(callback);
  }

  private emit(event: GestureEvent): void {
    for (const listener of this.listeners) listener(event);
  }

  private onTouchStart(e: TouchEvent): void {
    e.preventDefault();

    if (e.touches.length === 1) {
      const touch = e.touches[0];
      this.touchStart = { x: touch.clientX, y: touch.clientY, time: Date.now() };

      // Start long-press timer
      this.longPressTimer = setTimeout(() => {
        if (this.touchStart) {
          this.emit({
            type: "long-press",
            screenX: this.touchStart.x,
            screenY: this.touchStart.y,
          });
          this.touchStart = null; // consume the touch
        }
      }, 500);
    } else if (e.touches.length === 2) {
      this.cancelLongPress();
      const dx = e.touches[0].clientX - e.touches[1].clientX;
      const dy = e.touches[0].clientY - e.touches[1].clientY;
      this.initialPinchDistance = Math.hypot(dx, dy);
    }
  }

  private onTouchMove(e: TouchEvent): void {
    e.preventDefault();

    if (e.touches.length === 1 && this.touchStart) {
      const touch = e.touches[0];
      const dx = touch.clientX - this.touchStart.x;
      const dy = touch.clientY - this.touchStart.y;

      if (Math.hypot(dx, dy) > 10) {
        this.cancelLongPress();
        this.emit({
          type: "drag",
          screenX: touch.clientX,
          screenY: touch.clientY,
          deltaX: dx,
          deltaY: dy,
        });
        // Update start for continuous drag
        this.touchStart.x = touch.clientX;
        this.touchStart.y = touch.clientY;
      }
    } else if (e.touches.length === 2 && this.initialPinchDistance !== null) {
      const dx = e.touches[0].clientX - e.touches[1].clientX;
      const dy = e.touches[0].clientY - e.touches[1].clientY;
      const currentDistance = Math.hypot(dx, dy);
      const scale = currentDistance / this.initialPinchDistance;
      const cx = (e.touches[0].clientX + e.touches[1].clientX) / 2;
      const cy = (e.touches[0].clientY + e.touches[1].clientY) / 2;

      this.emit({ type: "pinch", screenX: cx, screenY: cy, scale });
      this.initialPinchDistance = currentDistance;
    }
  }

  private onTouchEnd(e: TouchEvent): void {
    this.cancelLongPress();

    if (this.touchStart && e.changedTouches.length === 1) {
      const touch = e.changedTouches[0];
      const elapsed = Date.now() - this.touchStart.time;
      const moved = Math.hypot(
        touch.clientX - this.touchStart.x,
        touch.clientY - this.touchStart.y
      );

      if (elapsed < 200 && moved < 10) {
        // Check for double-tap
        if (
          this.lastTap &&
          Date.now() - this.lastTap.time < 300 &&
          Math.hypot(touch.clientX - this.lastTap.x, touch.clientY - this.lastTap.y) < 20
        ) {
          this.emit({ type: "double-tap", screenX: touch.clientX, screenY: touch.clientY });
          this.lastTap = null;
        } else {
          this.emit({ type: "tap", screenX: touch.clientX, screenY: touch.clientY });
          this.lastTap = { x: touch.clientX, y: touch.clientY, time: Date.now() };
        }
      }
    }

    this.touchStart = null;
    this.initialPinchDistance = null;
  }

  private onTouchCancel(): void {
    this.cancelLongPress();
    this.touchStart = null;
    this.initialPinchDistance = null;
  }

  private cancelLongPress(): void {
    if (this.longPressTimer !== null) {
      clearTimeout(this.longPressTimer);
      this.longPressTimer = null;
    }
  }

  dispose(): void {
    this.canvas.removeEventListener("touchstart", this.onTouchStart);
    this.canvas.removeEventListener("touchmove", this.onTouchMove);
    this.canvas.removeEventListener("touchend", this.onTouchEnd);
    this.canvas.removeEventListener("touchcancel", this.onTouchCancel);
    this.listeners = [];
  }
}
```

### 8.2 Hex Picking

Convert a screen tap into a hex coordinate using Babylon.js scene picking and the hex layout transform:

```typescript
function screenToHex(scene: Scene, layout: HexLayout, screenX: number, screenY: number): AxialCoord | null {
  // Option A: Ray pick against a ground plane
  const pickResult = scene.pick(screenX, screenY, (mesh) => mesh.name === "ground");
  if (!pickResult?.hit || !pickResult.pickedPoint) return null;

  // The picked point is in world space. The ground plane is at y=0.
  const worldX = pickResult.pickedPoint.x;
  const worldZ = pickResult.pickedPoint.z; // z maps to our hex y-axis

  return pixelToHex(layout, worldX, worldZ);
}
```

### 8.3 Action Confirmation (Two-Tap Pattern)

To prevent accidental actions on a small touchscreen:

1. **First tap** on a target hex: Highlight the target, show a damage/hit-chance preview panel.
2. **Second tap** on the same hex OR tap the "Confirm" button: Execute the action.
3. **Tap elsewhere**: Cancel the selection, return to `SelectAction` state.

This two-step confirmation applies to attack actions and skill usage. Movement uses a single tap on the destination hex (since movement is undoable before the turn ends).

## 9. Save System

### 9.1 Storage Backend

**IndexedDB** via `idb-keyval` for all save data. IndexedDB supports multi-megabyte values, works offline, and persists across browser sessions. `localStorage` is avoided because it has a ~5 MB limit per origin and is synchronous (blocks the main thread).

```typescript
// src/save/SaveManager.ts

import { get, set, del, keys } from "idb-keyval";

const SAVE_KEY_PREFIX = "itactics_save_";
const AUTOSAVE_KEY = "itactics_autosave";

export class SaveManager {
  /** Save to a named slot. */
  async saveToSlot(slot: number, data: SaveData): Promise<void> {
    const serialized = SaveSerializer.serialize(data);
    await set(`${SAVE_KEY_PREFIX}${slot}`, serialized);
  }

  /** Load from a named slot. */
  async loadFromSlot(slot: number): Promise<SaveData | null> {
    const raw = await get(`${SAVE_KEY_PREFIX}${slot}`);
    if (!raw) return null;
    return SaveSerializer.deserialize(raw);
  }

  /** Auto-save (overwrites previous auto-save). */
  async autoSave(data: SaveData): Promise<void> {
    const serialized = SaveSerializer.serialize(data);
    await set(AUTOSAVE_KEY, serialized);
  }

  /** List all save slots with metadata (for the load screen). */
  async listSlots(): Promise<SaveSlotInfo[]> {
    const allKeys = await keys();
    const saveKeys = allKeys.filter((k) =>
      typeof k === "string" && k.startsWith(SAVE_KEY_PREFIX)
    );

    const slots: SaveSlotInfo[] = [];
    for (const key of saveKeys) {
      const raw = await get(key);
      if (raw) {
        const meta = SaveSerializer.extractMetadata(raw);
        slots.push(meta);
      }
    }
    return slots;
  }

  /** Delete a save slot. */
  async deleteSlot(slot: number): Promise<void> {
    await del(`${SAVE_KEY_PREFIX}${slot}`);
  }
}
```

### 9.5 Save Versioning and Migration

Every save has a `version` field. When loading, if the version is older than the current, the `SaveMigrator` applies sequential transforms:

```typescript
// src/save/SaveMigrator.ts

type MigrationFn = (data: Record<string, unknown>) => Record<string, unknown>;

const migrations: Record<number, MigrationFn> = {
  // Version 1 -> 2: added fatigue.max field
  2: (data) => {
    for (const brother of data.roster as any[]) {
      brother.fatigue.max ??= 100;
    }
    data.version = 2;
    return data;
  },
  // Version 2 -> 3: renamed "morale" to "resolve" in stats
  3: (data) => {
    for (const brother of data.roster as any[]) {
      brother.stats.resolve ??= brother.stats.morale ?? 50;
      delete brother.stats.morale;
    }
    data.version = 3;
    return data;
  },
};

export function migrate(data: Record<string, unknown>, targetVersion: number): Record<string, unknown> {
  let current = data.version as number;
  while (current < targetVersion) {
    const next = current + 1;
    const fn = migrations[next];
    if (!fn) throw new Error(`No migration from version ${current} to ${next}`);
    data = fn(data);
    current = next;
  }
  return data;
}
```

## 10. Audio System

### 10.1 Architecture

The audio system is built on Babylon.js `Sound` and `Music` classes, which wrap the Web Audio API. The system manages three categories of audio with independent volume controls.

```typescript
// src/audio/AudioManager.ts

export class AudioManager {
  private masterVolume = 1.0;
  private musicVolume = 0.7;
  private sfxVolume = 1.0;
  private ambientVolume = 0.5;
  private unlocked = false;

  private musicPlayer: MusicPlayer;
  private sfxPlayer: SFXPlayer;

  constructor(scene: Scene) {
    this.musicPlayer = new MusicPlayer(scene, this);
    this.sfxPlayer = new SFXPlayer(scene, this);
  }

  getEffectiveVolume(category: "music" | "sfx" | "ambient"): number {
    const catVol = category === "music" ? this.musicVolume
                 : category === "sfx" ? this.sfxVolume
                 : this.ambientVolume;
    return this.masterVolume * catVol;
  }

  setMasterVolume(v: number): void { this.masterVolume = clamp(v, 0, 1); this.updateAllVolumes(); }
  setMusicVolume(v: number): void { this.musicVolume = clamp(v, 0, 1); this.updateAllVolumes(); }
  setSFXVolume(v: number): void { this.sfxVolume = clamp(v, 0, 1); }
  setAmbientVolume(v: number): void { this.ambientVolume = clamp(v, 0, 1); this.updateAllVolumes(); }

  private updateAllVolumes(): void {
    this.musicPlayer.setVolume(this.getEffectiveVolume("music"));
  }
}
```

### 10.3 Music Crossfade

When transitioning between game states (e.g., overworld to combat), music tracks crossfade over 2 seconds:

```typescript
// src/audio/MusicPlayer.ts

export class MusicPlayer {
  private currentTrack: Sound | null = null;
  private scene: Scene;
  private audioManager: AudioManager;

  constructor(scene: Scene, audioManager: AudioManager) {
    this.scene = scene;
    this.audioManager = audioManager;
  }

  /** Crossfade to a new track over `duration` seconds. */
  async crossfadeTo(trackUrl: string, duration = 2.0): Promise<void> {
    const targetVolume = this.audioManager.getEffectiveVolume("music");

    const newTrack = new Sound("music", trackUrl, this.scene, null, {
      loop: true,
      autoplay: true,
      volume: 0,
    });

    const oldTrack = this.currentTrack;
    this.currentTrack = newTrack;

    // Animate volumes over `duration` seconds
    const steps = 30;
    const interval = (duration * 1000) / steps;

    for (let i = 1; i <= steps; i++) {
      await delay(interval);
      const t = i / steps;
      newTrack.setVolume(t * targetVolume);
      if (oldTrack) {
        oldTrack.setVolume((1 - t) * targetVolume);
      }
    }

    oldTrack?.dispose();
  }

  setVolume(v: number): void {
    if (this.currentTrack) this.currentTrack.setVolume(v);
  }

  stop(): void {
    this.currentTrack?.stop();
    this.currentTrack?.dispose();
    this.currentTrack = null;
  }
}

function delay(ms: number): Promise<void> {
  return new Promise((resolve) => setTimeout(resolve, ms));
}
```

### 11.7 Data Loader

```typescript
// src/data/DataLoader.ts

export class DataLoader {
  private cache = new Map<string, unknown[]>();

  async load<T>(path: string): Promise<T[]> {
    const cached = this.cache.get(path);
    if (cached) return cached as T[];

    const response = await fetch(`assets/data/${path}`);
    if (!response.ok) throw new Error(`Failed to load data: ${path}`);

    const data = await response.json();
    this.cache.set(path, data);
    return data as T[];
  }

  async loadWeapons(): Promise<WeaponData[]> { return this.load("weapons.json"); }
  async loadArmor(): Promise<ArmorData[]> { return this.load("armor.json"); }
  async loadBackgrounds(): Promise<BackgroundData[]> { return this.load("backgrounds.json"); }
  async loadPerks(): Promise<PerkData[]> { return this.load("perks.json"); }
  async loadEnemies(): Promise<EnemyData[]> { return this.load("enemies.json"); }
  async loadTerrain(): Promise<TerrainData[]> { return this.load("terrain.json"); }
  async loadEvents(): Promise<EventData[]> { return this.load("events.json"); }
  async loadContracts(): Promise<ContractData[]> { return this.load("contracts.json"); }

  /** Build a lookup map by ID for O(1) access. */
  async loadAsMap<T extends { id: string }>(path: string): Promise<Map<string, T>> {
    const items = await this.load<T>(path);
    const map = new Map<string, T>();
    for (const item of items) {
      map.set(item.id, item);
    }
    return map;
  }
}
```

### 12.2 Service Worker

The service worker uses Workbox for a hybrid caching strategy:

```typescript
// sw.ts (compiled by workbox-cli or vite-plugin-pwa)

import { precacheAndRoute } from "workbox-precaching";
import { registerRoute } from "workbox-routing";
import { CacheFirst, NetworkFirst } from "workbox-strategies";
import { ExpirationPlugin } from "workbox-expiration";

// Precache all build outputs (JS, CSS, HTML)
precacheAndRoute(self.__WB_MANIFEST);

// Cache-first for static assets (sprites, tiles, audio, data JSON)
registerRoute(
  ({ request }) =>
    request.destination === "image" ||
    request.url.includes("/assets/audio/") ||
    request.url.includes("/assets/data/"),
  new CacheFirst({
    cacheName: "static-assets",
    plugins: [
      new ExpirationPlugin({
        maxEntries: 500,
        maxAgeSeconds: 30 * 24 * 60 * 60, // 30 days
      }),
    ],
  })
);

// Network-first for API calls (cloud saves)
registerRoute(
  ({ url }) => url.pathname.startsWith("/api/"),
  new NetworkFirst({
    cacheName: "api-responses",
    networkTimeoutSeconds: 5,
  })
);
```

### 12.3 Cache Strategy Summary

| Resource Type | Strategy | Rationale |
|--------------|----------|-----------|
| JS/CSS/HTML bundles | Precache (build-time) | Versioned by hash, always available offline |
| Sprite atlases | Cache-first | Large files that rarely change |
| Tile textures | Cache-first | Rarely change |
| Audio files | Cache-first | Large, immutable after release |
| JSON data files | Cache-first | Change only with game updates |
| Cloud save API | Network-first | Prefer fresh data, fall back to cache |
| Analytics/telemetry | Network-only | Non-essential, skip if offline |

## Appendix B: Seeded PRNG

All randomness in the game uses a seeded pseudorandom number generator so that campaigns are deterministic given the same seed. This enables:

- Reproducible bug reports (share the seed)
- Ironman mode (no save-scumming changes outcomes)
- Replay functionality (future feature)

```typescript
// src/utils/RNG.ts

/**
 * Mulberry32 -- a fast 32-bit seeded PRNG.
 * Period: 2^32. Passes BigCrush statistical tests.
 */
export class RNG {
  private state: number;

  constructor(seed: number) {
    this.state = seed | 0;
  }

  /** Return a float in [0, 1). */
  next(): number {
    this.state = (this.state + 0x6d2b79f5) | 0;
    let t = Math.imul(this.state ^ (this.state >>> 15), 1 | this.state);
    t = (t + Math.imul(t ^ (t >>> 7), 61 | t)) ^ t;
    return ((t ^ (t >>> 14)) >>> 0) / 4294967296;
  }

  /** Return an integer in [min, max] inclusive. */
  nextInt(min: number, max: number): number {
    return min + Math.floor(this.next() * (max - min + 1));
  }

  /** Return true with probability `chance` (0-100). */
  rollPercent(chance: number): boolean {
    return this.nextInt(1, 100) <= chance;
  }

  /** Pick a random element from an array. */
  pick<T>(array: readonly T[]): T {
    return array[this.nextInt(0, array.length - 1)];
  }

  /** Shuffle an array in-place (Fisher-Yates). */
  shuffle<T>(array: T[]): T[] {
    for (let i = array.length - 1; i > 0; i--) {
      const j = this.nextInt(0, i);
      [array[i], array[j]] = [array[j], array[i]];
    }
    return array;
  }

  /** Save PRNG state for serialization. */
  getState(): number {
    return this.state;
  }

  /** Restore PRNG state from a save. */
  setState(state: number): void {
    this.state = state;
  }
}
```

## Appendix C: Event Bus

The event bus decouples systems that need to communicate without direct dependencies. Events are strongly typed.

```typescript
// src/core/EventBus.ts

type EventMap = {
  "unit:died": { entityId: string; killerEntityId: string | null };
  "unit:morale-changed": { entityId: string; oldState: string; newState: string };
  "combat:round-start": { roundNumber: number };
  "combat:battle-end": { result: "victory" | "defeat" | "retreat" };
  "overworld:day-advanced": { day: number };
  "overworld:settlement-entered": { settlementId: string };
  "contract:completed": { contractId: string; success: boolean };
  "save:auto-save-triggered": {};
};

type EventCallback<K extends keyof EventMap> = (data: EventMap[K]) => void;

export class EventBus {
  private listeners = new Map<string, Set<Function>>();

  on<K extends keyof EventMap>(event: K, callback: EventCallback<K>): void {
    let set = this.listeners.get(event);
    if (!set) {
      set = new Set();
      this.listeners.set(event, set);
    }
    set.add(callback);
  }

  off<K extends keyof EventMap>(event: K, callback: EventCallback<K>): void {
    this.listeners.get(event)?.delete(callback);
  }

  emit<K extends keyof EventMap>(event: K, data: EventMap[K]): void {
    const set = this.listeners.get(event);
    if (!set) return;
    for (const callback of set) {
      (callback as EventCallback<K>)(data);
    }
  }

  /** Remove all listeners. Call on scene disposal. */
  clear(): void {
    this.listeners.clear();
  }
}
```

This architecture is designed to be modular, testable, and performant on mobile browsers. Each system can be developed and tested independently. The data-driven approach means game designers can tune balance and add content by editing JSON files without touching code. The ECS pattern ensures new entity types and behaviors can be composed without modifying existing systems.
